1.Описание ошибки: Метод contains в классе StringUtils возвращает неверный результат для некоторых негативных тестов.

Шаги для воспроизведения:

Запустите тесты для метода contains с входными данными:
input_str = "", symbol = ""
input_str = "SkyPro", symbol = ""
input_str = "SkyPro", symbol = "z"
Ожидаемый результат:

Для входных данных ("", "", False) метод должен вернуть False.
Для входных данных ("SkyPro", "", False) метод должен вернуть False.
Для входных данных ("SkyPro", "z", False) метод должен вернуть False.
Фактический результат:

Для всех тестов метод возвращает True, что не соответствует ожидаемому результату.

Пожалуйста, проверьте логику метода contains и исправьте его, чтобы он корректно обрабатывал указанные негативные сценарии.



2.Описание ошибки: Ошибка в методе is_empty
Описание ошибки: Метод is_empty в классе StringUtils возвращает неверный результат для строк, содержащих пробелы, 
табуляцию и перевод строки. Ожидается, что такие строки будут считаться пустыми после удаления пробелов, 
но метод возвращает False вместо ожидаемого True.

Шаги для воспроизведения:

Запустите тесты для метода is_empty с входными данными:

input_str = " \t\n" (строка содержит пробелы, табуляцию и перевод строки)
Ожидаемый результат:

Метод должен вернуть True, так как строка считается пустой после триммирования.
Фактический результат:

Метод возвращает False, что не соответствует ожидаемому результату.
Пожалуйста, проверьте реализацию метода is_empty и убедитесь, что он корректно обрабатывает строки, 
содержащие только пробелы, табуляцию и переводы строк, удаляя их перед проверкой на пустоту.
